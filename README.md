# 🔒 מפעיל סיסמאות אקראיות

פרויקט פשוט ליצירת סיסמאות אקראיות וחזקות.

## 📋 תיאור

אפליקציית Web פשוטה וידידותית ליצירת סיסמאות אקראיות וחזקות. מושלם ללימוד JavaScript בסיסי!

### ✨ תכונות

- 🎚️ **בחירת אורך** - החלק מחוון בין 4 ל-32 תווים
- 🔤 **אותיות קטנות** - a-z (26 תווים)
- 🔠 **אותיות גדולות** - A-Z (26 תווים)
- 🔢 **מספרים** - 0-9 (10 תווים)
- 🔣 **סימנים מיוחדים** - !@#$%^&*() ועוד (29 תווים)
- ⚡ **עדכון מיידי** - הסיסמה מתעדכנת בזמן אמת
- 📱 **Responsive** - עובד מצוין גם בנייד

## 🚀 איך להשתמש

## 🎯 אתגר בונוס - סיסמאות חזקות יותר

### הבעיה

בגרסה הבסיסית (`main.js`), אם המשתמש בחר את כל 4 סוגי התווים, יכול להיות שהסיסמה לא תכלול את כל הסוגים. 

**דוגמה:** סיסמה באורך 12 תווים עם כל התיבות מסומנות עדיין יכולה להיות `"aaaaaaaaaaaaa"` - בלי אותיות גדולות, מספרים או סימנים!

### הפתרון - אלגוריתם משופר

הקובץ `bonus.js` מיישם אלגוריתם בן 3 שלבים:

1. **ערובה לכיסוי**: מוסיף לפחות תו אחד מכל סוג שנבחר
2. **מילוי**: משלים את יתר התווים באופן אקראי
3. **ערבוב**: מערבב את הסיסמה כדי שלא תהיה בת ניבוי

### משימות התרגיל

התלמידים צריכים להשלים 5 משימות ב-`bonus.js`:

| משימה | תיאור | רמז |
|-------|-------|-----|
| 1️⃣ | `getRandomChar()` - בחירת תו אקראי | השתמש ב-`Math.random()` ו-`Math.floor()` |
| 2️⃣ | הוספת קטגוריות למערך ולמחרוזת | השתמש ב-`push()` ו-`+=` |
| 3️⃣ | לולאה להוספת תו מכל קטגוריה | לולאת `for` עם `getRandomChar()` |
| 4️⃣ | לולאה למילוי יתר התווים | לולאה מ-`password.length` עד `length` |
| 5️⃣ | ערבוב הסיסמה | `split()` → `sort()` → `join()` |

### איך להשתמש בקובץ הבונוס

1. פתח את `index.html`
2. שנה את השורה `<script src="main.js"></script>` ל:
   ```html
   <script src="bonus.js"></script>
   ```
3. השלם את המקומות המסומנים ב-`// TODO:` בקוד
4. בדוק את הפתרון שלך - נסה ליצור סיסמאות ווודא שכל סוגי התווים מופיעים
5. אם נתקעת, בדוק את `bonus-solution.js`

### איך לבדוק שהפתרון עובד?

צור סיסמה עם כל 4 הקטגוריות באורך 12 תווים ובדוק שיש:
- ✓ לפחות אות קטנה אחת
- ✓ לפחות אות גדולה אחת
- ✓ לפחות מספר אחד
- ✓ לפחות סימן אחד

### שאלות לחשיבה

1. **למה צריך ערבוב?** מה היה קורה אם לא היינו מערבבים את הסיסמה?
2. **האם `Math.random()` מספיק בטוח?** האם זה מתאים לסיסמאות אמיתיות?
3. **איך נוכל לשפר?** איזה אלגוריתם ערבוב טוב יותר?
